## Ответы на контрольные вопросы к лабораторной работе №3

**1. Что такое целевая аудитория (ЦА)?**

Целевая аудитория (ЦА) — это определенная группа людей с **общими характеристиками, желаниями, потребностями, образом жизни**, которые позволяют им **примерно одинаково взаимодействовать с продуктом**. Это люди, которые, скорее всего, заинтересуются разрабатываемым продуктом, услугой или сервисом.

**2. Какие есть виды целевой аудитории?**

* **По классу конечных потребителей:**
    * **B2B** (business to business) — компания продаёт свой товар или услуги другим компаниям.
    * **B2C** (business to customer) — компания продаёт товар или услуги физическим лицам.
    * **B2G** (business to government) — компания продаёт товары или услуги государственным учреждениям и организациям.
* **По масштабу:**
    * **Широкая.**
    * **Узкая.**

**3. Что такое User Persona?**

User Persona (персонаж пользователя) — это **воображаемый, но реалистичный, собирательный, но детальный образ пользователя** с суммой его основных потребностей, характеристик и целей. Это типичный представитель сегмента, поэтому **каждому сегменту должна соответствовать хотя бы один персонаж**.

**4. Из чего состоит User Persona?**

* **Имя.**
* **Пол.**
* **Возраст.**
* **Фото.**
* **Месторасположение (страна/город).**
* **Семейное положение.**
* **Род занятий.**
* **Степень «вхождения» в предметную область или опыт использования продукта.**
* **Контекст использования продукта.**
* **Цели и потребности взаимодействия.**
* **Проблемы взаимодействия.**
* **Дополнительно:** ожидания, навыки, мотивация.

**5. Что такое User Story?**

User Story (пользовательская история) — это **короткая история с описанием возможных вариантов применения продукта.** Это 1-2 предложения о том, чего хочет достичь персонаж, используя продукт, что пользователь может делать взаимодействую с продуктом.

**6. Как составить User Story?**

Использовать шаблон:

> **«Как кто-то (роль/персонаж) я хочу (цель/желание), чтобы (выгода).»**

**7. Что такое Use Case?**

Use Case (сценарий использования) — это **вымышленная история о том, как пользователь взаимодействует с продуктом, чтобы достичь определённых целей.**

**8. Из чего состоит Use Case?**

* **Название (Name).**
* **Краткое описание (Brief Description).**
* **Участники (Actors).**
* **Предусловия (Preconditions).**
* **Триггер (Trigger).**
* **Базовый сценарий (Basic Flow).**
* **Альтернативные сценарии (Alternative Flows).**
* **Исключительные сценарии (Exceptional Flows).**
* **Постусловие (Post Conditions).**

**9. Как связаны User Story и Use Case?**

* User Story описывает **потребность пользователя** и **желаемый результат**.
* Use Case описывает **последовательность действий**, которые пользователь должен выполнить, чтобы **достичь желаемого результата**.

**Use Case создаются на основе User Story.** Сценарии добавляют к истории **подробности взаимодействия с продуктом**. Use Case представляет список действия для того, чтобы **пользователь получил желаемого при взаимодействии с продуктом**.
